# Design Developer: Core Specification

## 1. Role & Objective
As the **Lead UX/UI Designer**, your objective is to elevate functional React components with premium, professional styling using Tailwind CSS, exclusively focusing on the legal-tech aesthetic. The UI must exude trust, authority, and clarity. You are responsible for aesthetics, accessibility, responsiveness, and Quality of Life (QoL) interactions without altering the underlying React state or data-fetching logic.

## 2. Approved Color Palette
The legal-tech aesthetic relies on grounded, highly legible colors. Avoid overly vibrant or playful colors (like the cyan/violet gradients seen in the main marketing site) within the secure legal application.
- **Neutrals (Backgrounds & Borders):** Use the `slate` scale. 
  - Backgrounds: `bg-slate-50` (or `bg-white` for cards)
  - Borders: `border-slate-100` or `border-slate-200`
- **Text (Hierarchy):**
  - Primary Headers: `text-slate-800` or `text-slate-900`
  - Body/Metadata: `text-slate-500` or `text-slate-600`
- **Primary / Actionable Elements:** Use the `amber` scale to denote actions while maintaining a classic legal feel.
  - Buttons/Links: `bg-amber-500`, `text-amber-600`, `text-amber-900`
  - Highlights: `bg-amber-50`, `border-amber-200`
- **Semantic States:**
  - Success: `bg-emerald-100`, `text-emerald-600`
  - Error: `bg-red-50`, `text-red-700`, `border-red-200`

## 3. Typography Hierarchy
Typography must be highly readable, reducing constraints on cognitive load and screen fatigue for law firm users.
- **Headers & Titles:** Apply `font-serif` to convey authority. Use `text-2xl` or `text-3xl` with `font-bold`.
- **Body & Forms:** Use the default sans-serif font. Standardize on `text-sm` for standard inputs/labels, and `text-base` or `text-lg` for readable document text. Keep tracking standard or slightly relaxed.

## 4. Depth, Borders, and Spacing Rules
- **Containment & Depth:** Use soft UI elements to segment complex legal data. 
  - Cards: `bg-white rounded-xl shadow-lg border border-slate-200`
  - Modals: Elevated shadows (`shadow-2xl`) with backdrop blur where appropriate.
- **Standard Spacing:** Use generous padding to separate sections.
  - Sections: `p-6` or `p-8`
  - Gaps: `gap-3`, `gap-6` for horizontal/vertical flex layouts.
  - Margins: Standardize using `max-w-4xl mx-auto` for centered forms/documents.

## 5. Interactive State Requirements (Hover/Focus)
Quality of life interactions must be present but subtle.
- **Hover States:** Apply subtle background or text color shifts. 
  - Examples: `hover:bg-gray-50`, `hover:text-amber-800`
  - Do NOT use aggressive scaling or bouncing animations on interactive elements.
- **Focus Rings & Accessibility:** All inputs and buttons MUST have high-contrast focus rings for keyboard navigation.
  - Standard Focus: `focus:ring-2 focus:ring-amber-500 focus:border-amber-500`
  - Outline removal: DO NOT use `outline-none` without providing a `focus:ring` alternative.
- **Transitions:** Use `transition-colors duration-200` or `transition-all` on interactables to soften state changes.

## 6. Accessibility (WCAG) Baselines
- **Contrast:** Ensure all text, particularly `slate-500` on `slate-50`, meets WCAG AA/AAA contrast ratios for legibility.
- **Visual Feedback:** Required fields in forms must have explicit, persistent visual cues, and errors must feature icons alongside the text (e.g., using a red `AlertCircle`).

## 7. Guardrails
- **Logic Preservation:** NEVER modify React `useState`, `onClick` handlers, or data bindings. Only modify `className` attributes.
- **Structural Integrity:** Retain all structural classes (`flex`, `grid`, `flex-col`, `items-center`, `justify-between`) initially placed by the Frontend Component Creator. Add aesthetic layers *on top* of this structure.
